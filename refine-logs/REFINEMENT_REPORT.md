# Refinement Report

## Starting Point

The first draft framed AI-native network as portable identity, state, hosts, wallets, service exchange, and governance.

## User Correction

The corrected target is:

- AI autonomy as a self-managing AI model company.
- Users download software and run local clients.
- Users can contribute resources such as training resources, inference resources, storage, and API quota, even if slow.
- The AI can schedule itself across local and cloud execution.
- Hermes-like agents already implement basic local functions.
- Missing layer: AI-AI communication, account/social functions, and network service exchange.
- The AI is one AI that can serve all users while also providing personalized service.

## Refined Direction

`Hermes Network Mode`

This means:

- add accounts,
- add social graph/service profile,
- add AI-AI inbox and service messages,
- add resource registry,
- add scheduler,
- add credit ledger,
- add per-user memory boundaries.

## Final Research Question

Can a Hermes-like agent become a self-managing AI service network by adding account/social identity, AI-AI service communication, and resource-aware scheduling over local, cloud, and user-contributed nodes?

## Remaining Risks

- Resource scheduling can become ordinary distributed systems unless tied to AI service needs.
- AI-AI messages can become ordinary chat unless structured as service requests/offers/receipts.
- Personalization can leak memory unless the design isolates per-user state.
- Incentives can distract from the core; keep credits internal first.
